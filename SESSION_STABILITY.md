# Session Stability Notes

작성일: 2026-04-18

## 문제 요약

- 서버가 단일 세션에 과도하게 묶이면, 한 연결의 이상이 전체 응답 흐름처럼 보일 수 있다.
- handshake만으로는 장기 연결의 생존 여부를 보장할 수 없다.
- 작업 중/대기 중 상태가 섞이면, 끊긴 세션이 살아있는 것처럼 보이거나 반대로 정상 세션이 오탐 종료될 수 있다.

## 적용 원칙

1. 서버는 여러 세션을 동시에 받아도 각 세션을 독립적으로 관리한다.
2. handshake는 최초 연결 검증에만 사용한다.
3. 지속 생존 확인은 heartbeat와 last_seen 타임스탬프로 처리한다.
4. 연결 이상이 있으면 해당 세션만 종료하고 서버 프로세스는 유지한다.
5. 오래된 세션은 자동 만료한다.
6. 작업 실행과 연결 상태 관리는 분리한다.

## 현재 적용 내용

- `Sm_AIAgent/agent_api.py`
  - 세션별 `last_seen` 저장
  - 세션 접근 락 추가
  - idle timeout 기반 자동 정리
  - `/agent/sessions` 응답에 `last_seen_sec`와 timeout 정보 추가

## 운영 규칙

- 클라이언트는 연결 직후 handshake를 1회 수행한다.
- 이후에는 heartbeat 또는 요청/응답 흐름으로 상태를 갱신한다.
- 응답이 끊기면 전체 서버 재시작보다 해당 세션 cancel/reset을 우선한다.
- 서버가 죽지 않게 하려면, 네트워크 재연결보다 먼저 세션 격리와 예외 격리를 보장해야 한다.
