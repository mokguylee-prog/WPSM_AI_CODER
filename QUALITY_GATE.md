# Quality Gate

이 저장소의 변경은 다음 기준을 만족할 때만 "좋은 상태"로 본다.

## Gate Rules

- 총점 95점 이상이어야 한다.
- 서버는 세션 실패로 전체 offline처럼 보이면 안 된다.
- 클라이언트는 실패 원인을 사용자에게 설명해야 한다.
- 재연결 시 이전 세션 잔재가 새 작업을 방해하면 안 된다.
- 로그와 상태는 문제 재현과 원인 파악에 충분해야 한다.

## Improvement Loop

1. 현재 점수를 계산한다.
2. 95점 미만 항목을 찾는다.
3. 항목별로 수정한다.
4. 다시 계산한다.
5. 95점 이상이 될 때까지 반복한다.

## Non-Negotiables

- 서버가 죽지 않게 설계한다.
- 죽더라도 자동 복구 경로를 둔다.
- 실패를 숨기지 않는다.
- 실패 원인을 분류해서 보여준다.
