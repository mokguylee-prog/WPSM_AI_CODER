# C 코드 생성 실행 가이드

## StarCoder2의 코드 생성 방식

> **중요**: StarCoder2는 **코드 완성(Code Completion)** 모델입니다.
> 자연어 명령("함수 작성해줘")이 아닌 **부분 코드를 입력하면 나머지를 완성**합니다.

## FIM (Fill-in-the-Middle) 방식

코드 중간 부분을 채우는 특수 기능:

```python
# FIM 형식: <fim_prefix>앞부분<fim_suffix>뒷부분<fim_middle>
prompt = """<fim_prefix>#include <stdio.h>

int main() {
    int arr[] = {5, 3, 8, 1, 9, 2};
    int n = 6;
    <fim_suffix>
    printf("Sorted: ");
    for (int i = 0; i < n; i++)
        printf("%d ", arr[i]);
    return 0;
}<fim_middle>"""
```

→ StarCoder2가 중간 빈 부분에 정렬 코드를 삽입

## 생성 파라미터 설정

```python
outputs = model.generate(
    inputs,
    max_new_tokens=256,     # 생성할 최대 토큰 수 (길이 조절)
    temperature=0.2,         # 낮을수록 일관성, 높을수록 다양성 (0.0~1.0)
    top_p=0.95,             # nucleus sampling (0.9~0.99)
    top_k=50,               # top-k sampling
    do_sample=True,          # 샘플링 활성화
    repetition_penalty=1.1,  # 반복 방지
    pad_token_id=tokenizer.eos_token_id
)
```

| 파라미터 | 권장값 | 설명 |
|----------|--------|------|
| temperature | 0.1~0.3 | 코드는 낮은 값 사용 |
| max_new_tokens | 128~512 | 함수 하나에 128-256 충분 |
| top_p | 0.90~0.95 | 그대로 사용 |

## 생성 품질 향상 팁

1. **구체적인 시작 코드 제공**: 함수 시그니처, 변수명, 자료형을 명확히
2. **주석으로 의도 설명**: `// Sort in ascending order` 처럼 힌트 제공
3. **헤더 파일 포함**: `#include <stdlib.h>` 등을 미리 작성
4. **temperature 낮게**: C 코드는 0.1~0.2 권장
5. **여러 번 생성 후 선택**: do_sample=True로 여러 후보 생성
