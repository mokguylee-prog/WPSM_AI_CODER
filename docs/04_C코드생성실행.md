# 코드 생성 실행 가이드

## Sm_AICoder의 코드 생성 방식

> Sm_AICoder는 **Instruction-Following** 모델(Qwen2.5-Coder-Instruct)을 사용합니다.
> **자연어로 직접 코드를 요청**할 수 있습니다 (이전 StarCoder2의 코드 완성 방식과 다름).

---

## 기본 실행

```powershell
# 1. 서버 시작
.\\start_server.ps1

# 2. GUI 또는 CLI 클라이언트 실행
.\\start_gui.ps1
.\\start_client.ps1
```

---

## 프롬프트 작성법

### 자연어로 직접 요청 (권장)

```text
C 언어로 버블 정렬 구현해줘
팩토리얼을 구하는 재귀함수 만들어줘
이진 탐색 알고리즘을 C로 작성해줘, 에러 처리 포함
```

### 구체적 요청 (더 좋은 결과)

```text
C 언어로 링크드 리스트를 구현해줘.
- insert, delete, print 함수 포함
- main에서 테스트 코드도 작성해줘
- 메모리 해제까지 처리해줘
```

### 다중 턴 대화 활용

```text
[1턴] C 언어로 스택 구현해줘
[2턴] push, pop에 에러 처리 추가해줘
[3턴] 이걸 동적 배열 기반으로 바꿔줘
```

## REST API로 코드 생성

### 단발성 생성 (/generate)

```bash
curl -X POST http://localhost:8888/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "C 언어로 퀵 정렬 구현해줘", "max_tokens": 1024}'
```

### 다중 턴 대화 (/chat)

```bash
curl -X POST http://localhost:8888/chat \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "user", "content": "C 언어로 해시 테이블 구현해줘"},
      {"role": "assistant", "content": "...이전 응답..."},
      {"role": "user", "content": "충돌 처리를 체이닝 방식으로 바꿔줘"}
    ],
    "max_tokens": 1024
  }'
```

## 생성 파라미터

| 파라미터 | 기본값 | 설명 |
| -------- | ------ | ---- |
| `temperature` | 0.3 | 낮을수록 안정적, 높을수록 창의적 (0.0~1.0) |
| `max_tokens` | 1024 | 생성할 최대 토큰 수 |
| `top_p` | 0.95 | 다양성 제어 |

> 코드 생성에는 `temperature` 0.1~0.3을 권장합니다.

## 코드 요청 예시 모음

### 알고리즘

```text
C 언어로 버블 정렬 구현해줘
퀵 정렬 알고리즘을 C로 작성해줘
이진 탐색을 재귀 방식으로 구현해줘
피보나치 수열을 동적 프로그래밍으로 구현해줘
```

### 자료구조

```text
단일 링크드 리스트 구현해줘 (삽입, 삭제, 출력)
스택을 배열로 구현해줘
큐를 링크드 리스트로 구현해줘
이진 탐색 트리 구현해줘
```

### 문자열/파일

```text
문자열 뒤집기 함수 만들어줘
CSV 파일을 파싱하는 C 코드 작성해줘
파일에서 한 줄씩 읽어서 출력하는 코드 만들어줘
```

### 메모리 관리

```text
동적 배열(가변 크기 배열) 구현해줘
간단한 메모리 풀 할당자 만들어줘
```

## 생성 품질 향상 팁

1. **구체적으로 요청**: "C 언어로", "에러 처리 포함", "main 함수에서 테스트" 등
2. **한국어/영어 모두 가능**: 모델이 사용자 언어에 맞춰 응답
3. **다중 턴 활용**: 처음에 기본 코드 → 이어서 수정/개선 요청
4. **temperature 낮게**: 안정적인 코드가 필요하면 0.1~0.2
