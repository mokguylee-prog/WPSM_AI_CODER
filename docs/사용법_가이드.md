# Sm_AICoder 사용법 가이드

> 이 환경은 **GPU 없음 / CPU 전용** 구성입니다.
> Qwen2.5-Coder-7B-Instruct 모델에 자연어로 코드를 요청할 수 있습니다.

---

## 1. 환경 준비

```powershell
# 가상환경 활성화
venv\Scripts\activate
```

활성화되면 프롬프트 앞에 `(venv)` 표시가 나타납니다.

---

## 2. 핵심 개념

Sm_AICoder는 자연어 지시를 이해하는
Instruction-Following 코드 생성 모델입니다.

- 기존 코드 완성형 입력 예:
  `void bubble_sort(int arr[], int n) {`
- 현재 자연어 입력 예:
  `C 언어로 버블 정렬 구현해줘`
- 응답 언어:
  사용자가 한국어로 쓰면 한국어로,
  영어로 쓰면 영어로 답합니다.

---

## 3. 사용 방법

### 3-1. GUI 클라이언트

```powershell
.\start_gui.ps1
```

- 왼쪽 패널에서 자연어 요청을 입력합니다.
- `Ctrl+Enter`로 전송합니다.
- 오른쪽 패널은 코드 블록만 자동 추출합니다.
- 하단 패널은 설명 포함 전체 응답을 보여줍니다.
- 상단에서 온도와 최대 토큰 수를 조절할 수 있습니다.

### 3-2. CLI 클라이언트

```powershell
.\start_client.ps1
```

- 여러 줄을 입력한 뒤 빈 줄을 한 번 더 입력하면 전송됩니다.
- `:help`로 사용 가능한 명령을 볼 수 있습니다.
- `:clear`로 대화 히스토리를 초기화할 수 있습니다.
- `:save`로 마지막 응답을 `output.txt`에 저장할 수 있습니다.

### 3-3. 웹 대시보드

브라우저에서 <http://localhost:8888> 로 접속합니다.

- 서버 상태와 모델 정보를 확인할 수 있습니다.
- 업타임과 요청 통계를 볼 수 있습니다.
- 최근 요청 행을 클릭하면 상세 내용을 볼 수 있습니다.

### 3-4. REST API 직접 호출

```bash
# 단발성 코드 생성
curl -X POST http://localhost:8888/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "C 언어로 버블 정렬 구현해줘", "max_tokens": 512}'

# 다중 턴 대화
curl -X POST http://localhost:8888/chat \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "C 언어로 버블 정렬 구현해줘"}]}'

# 상태 확인
curl http://localhost:8888/health
```

## 4. 프롬프트 작성 팁

- 요청 언어는 한국어와 영어 모두 가능합니다.
- 목표 언어를 함께 쓰면 결과가 더 안정적입니다.
  예: `C 언어로`, `C++로`
- 요구사항을 구체적으로 적을수록 결과가 좋아집니다.
  예: `에러 처리 포함`, `함수 분리`, `주석 추가`
- 한 번에 큰 요구를 하기보다
  여러 턴으로 나눠 보완하는 방식이 좋습니다.

### 자주 조절하는 파라미터

| 파라미터 | 기본값 | 의미 |
| -------- | ------ | ---- |
| `temperature` | `0.3` | 낮을수록 안정적이고 높을수록 다양해집니다. |
| `max_tokens` | `1024` | 생성할 최대 토큰 수입니다. |
| `top_p` | `0.95` | 후보 다양성을 제어합니다. |

## 5. 빠른 참조 명령어

```powershell
# 서버 시작
.\start_server.ps1

# GUI 클라이언트 시작
.\start_gui.ps1

# CLI 클라이언트 시작
.\start_client.ps1

# 서버 종료
.\stop_server.ps1

# 환경 확인
venv\Scripts\python.exe server\scripts\check_env.py

# 모델 다운로드
venv\Scripts\python.exe server\scripts\download_model.py
```

## 6. 트러블슈팅

### 모델 파일을 찾을 수 없다는 오류

```text
GGUF 모델이 없습니다: .../Sm_AICoder/models/gguf
```

다음 명령으로 모델을 다운로드합니다.

```powershell
venv\Scripts\python.exe server\scripts\download_model.py
```

### 생성 속도가 느린 경우

- CPU 환경에서는 모델 로딩에 30~60초 정도 걸릴 수 있습니다.
- 응답 생성에도 수십 초가 걸릴 수 있습니다.
- 빠른 테스트가 목적이면 `max_tokens`를 줄여 보세요.

### 포트가 이미 사용 중인 경우

```powershell
netstat -ano | findstr :8888
taskkill /PID <pid> /F
```
