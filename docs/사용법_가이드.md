---

## 2. 핵심 개념: Instruction-Following 모델

Sm_AICoder는 **자연어 지시를 이해하는 Instruction-Following 모델**입니다.
이전 StarCoder2(코드 완성 전용)와 달리, 자연어로 직접 코드를 요청할 수 있습니다.

| 이전 (StarCoder2 — 코드 완성) | 현재 (Sm_AICoder — 자연어 지시) |
| ---- | ---- |
| `"void bubble_sort(int arr[], int n) {\n    for (int i = 0;"` | `"C 언어로 버블 정렬 구현해줘"` |
| `"int factorial(int n) {\n    if (n <= 1)\n        return 1;\n"` | `"팩토리얼 재귀함수 만들어줘"` |

## 3. 사용 방법

### 3-1. GUI 클라이언트 (권장)

```powershell
.\start_gui.ps1
```

- ① 왼쪽 패널: 자연어로 코드 요청 입력 (Ctrl+Enter 전송)
- ② 오른쪽 패널: 코드 블록만 자동 추출, 클립보드 복사
- ③ 하단 패널: 전체 AI 응답 (설명 + 코드)
- 상단: 온도/최대 토큰 조절, 대화 히스토리 초기화

### 3-2. CLI 클라이언트

```powershell
.\start_client.ps1
```

- 자연어로 코드 요청 입력
- 여러 줄 입력 후 빈 줄로 전송
- `:help` 로 명령어 확인
- 다중 턴 대화 히스토리 유지

### 3-3. 웹 대시보드

브라우저에서 `http://localhost:8888` 접속

- 서버 상태 / 모델 정보 / 업타임
- 요청 통계 (총 요청수, 토큰 합계, 평균 응답 시간)
- 최근 요청 히스토리 (행 클릭 시 상세 보기)

### 3-4. REST API 직접 호출

```bash
# 단발성 코드 생성
curl -X POST http://localhost:8888/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "C 언어로 버블 정렬 구현해줘", "max_tokens": 512}'

# 다중 턴 대화
curl -X POST http://localhost:8888/chat \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "C 언어로 버블 정렬 구현해줘"}]}'

# 상태 확인
curl http://localhost:8888/health
```

## 4. 프롬프트 작성 팁

- 한국어/영어 모두 가능 (사용자 언어에 맞춰 응답)
- 구체적으로 요청할수록 좋은 결과 (`"C 언어로"`, `"에러 처리 포함해서"` 등)
- 다중 턴을 활용해 단계적으로 요청 가능

**설정값 조절:**

| 파라미터 | 기본값 | 의미 |
| ---- | ---- | ---- |
| `temperature` | 0.3 | 낮을수록 안정적, 높을수록 창의적 (0.0~1.0) |
| `max_tokens` | 1024 | 생성할 최대 토큰 수 |
| `top_p` | 0.95 | 다양성 제어 |

## 6. 트러블슈팅

### 모델 파일을 찾을 수 없다는 오류

```
GGUF ��델이 없습니다: .../Sm_AICoder/models/gguf
```

→ `venv\Scripts\python.exe scripts\download_model.py` 실행

### 생성 속도가 느린 경우

CPU 환경이므로 모델 로딩에 30~60초, 응답 생성에 수십 초 소요됩니다.
`max_tokens` 값을 줄여서 빠르게 테스트할 수 있습니다.

### 포트 충돌

```powershell
netstat -ano | findstr :8888
```
