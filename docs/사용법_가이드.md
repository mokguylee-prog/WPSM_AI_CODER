# StarCoder2 사용법 가이드

> 이 환경은 **GPU 없음 / CPU 전용** 구성입니다.  
> 모든 모델 파일과 패키지는 **D 드라이브**에 설치되어 있습니다.

---

## 1. 환경 준비 (매 세션 시작 전)

PowerShell 또는 CMD에서 가상환경을 활성화합니다.

```powershell
D:\StarCoder2\venv\Scripts\activate
```

활성화되면 프롬프트 앞에 `(venv)` 표시가 나타납니다.

---

## 2. 핵심 개념: 이 모델은 "코드 완성" 모델입니다

StarCoder2는 **자연어 명령을 이해하는 모델이 아닙니다.**  
반드시 **부분 C 코드**를 프롬프트로 제공해야 모델이 나머지를 채워줍니다.

| 잘못된 방법 (자연어) | 올바른 방법 (부분 코드) |
|---|---|
| `"버블 정렬 함수 만들어줘"` | `"void bubble_sort(int arr[], int n) {\n    for (int i = 0;"` |
| `"팩토리얼 재귀함수"` | `"int factorial(int n) {\n    if (n <= 1)\n        return 1;\n"` |

## 3. 주요 스크립트 사용법

### 3-1. 기본 C 코드 생성 (가장 자주 쓰는 방법)

```bash
cd d:/work_web/StarCoder
python scripts/generate_cpu_gguf.py
```

**동작 방식:**
- `D:/StarCoder2/models/gguf/starcoder2-3b-Q4_K_M.gguf` 모델을 자동으로 로드
- 스크립트 내부에 미리 정의된 3가지 예시(버블 정렬, 팩토리얼, 이진 탐색)를 순서대로 생성
- 각 예시의 프롬프트 + 생성된 코드를 화면에 출력

**출력 예시:**
```
=== StarCoder2 CPU 코드 생성 ===
모델: starcoder2-3b-Q4_K_M.gguf
로딩 중...

=======================================================
[예시 1] 버블 정렬

[프롬프트]
void bubble_sort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {

[생성 결과]
void bubble_sort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                int tmp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = tmp;
            }
        }
    }
}
```

### 3-2. 나만의 프롬프트로 코드 생성하기

`scripts/generate_cpu_gguf.py` 파일을 열어 `EXAMPLES` 리스트를 수정합니다.

```python
EXAMPLES = [
    {
        "desc": "내가 원하는 함수 설명",
        "prompt": """\
#include <stdio.h>

// 내가 원하는 함수의 시작 부분
int my_function(int x) {
    """,
    },
]
```

**프롬프트 작성 팁:**
- `#include` 헤더를 포함하면 더 정확한 코드가 생성됩니다
- 영어 주석(`// comment`)을 사용하면 품질이 높아집니다
- 함수 시그니처까지는 직접 써주고, 함수 본문부터 채우게 하면 좋습니다
- 파라미터 이름을 설명적으로 작성하면(예: `filename`, `count`) 더 잘 이해합니다

**설정값 조정:**

| 파라미터 | 기본값 | 의미 |
|---|---|---|
| `MAX_TOKENS` | 256 | 생성할 최대 토큰 수. 길게 생성하려면 늘리세요 |
| `TEMPERATURE` | 0.2 | 낮을수록 안정적, 높을수록 창의적 (0.0~1.0) |
| `TOP_P` | 0.95 | 다양성 제어. 보통 건드리지 않아도 됩니다 |

### 3-4. 코드 중간 채우기 (FIM - Fill-in-the-Middle)

코드의 **앞부분과 뒷부분**을 알고 있고, **중간**을 채우고 싶을 때 사용합니다.

> **주의**: `fim_completion.py`는 GPU 버전(starcoder2-7b)으로 작성되어 있어 CPU 환경에서 그대로 실행하면 느립니다. GGUF 버전으로 수정이 필요합니다.

**FIM 개념:**
```
앞부분 (prefix) + [???중간???] + 뒷부분 (suffix)
                 ↑ 이 부분을 모델이 채워줌
```

**프롬프트 구조:**
```
<fim_prefix>앞부분 코드<fim_suffix>뒷부분 코드<fim_middle>
```

**사용 예시 (개념):**
```c
// prefix (앞부분)
int main() {
    int arr[] = {64, 34, 25, 12};
    int n = 4;
    // Sort the array

// suffix (뒷부분)
    printf("Sorted\n");
    return 0;
}

// → 모델이 중간의 정렬 코드를 생성
```

### 3-5. HTTP API 서버로 사용하기

> **주의**: `api_server.py`는 GPU 버전 기준 코드입니다. CPU 환경에서는 매우 느릴 수 있습니다.

```bash
python scripts/api_server.py
```

서버 시작 후 포트 8888에서 HTTP 요청을 받습니다.

**코드 생성 요청:**
```bash
curl -X POST http://localhost:8888/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "int factorial(int n) {\n    if (n <= 1)\n        return 1;\n", "max_tokens": 128}'
```

**상태 확인:**
```bash
curl http://localhost:8888/health
```

**응답 형식:**
```json
{
  "generated": "int factorial(int n) {\n    if (n <= 1)\n        return 1;\n    return n * factorial(n - 1);\n}",
  "prompt_tokens": 23,
  "generated_tokens": 15
}
```

## 5. 트러블슈팅

### 모델 파일을 찾을 수 없다는 오류

```
모델 파일을 찾을 수 없습니다: D:/StarCoder2/models/gguf
먼저 실행하세요: python scripts/download_model.py
```

→ `python scripts/download_model.py` 실행하여 모델을 다시 다운로드합니다.

### `llama_cpp` 모듈 없다는 오류

```
설치 필요: pip install llama-cpp-python
```

→ 가상환경 활성화 후 실행:
```bash
pip install llama-cpp-python
```

### 생성 속도가 너무 느린 경우

CPU 환경이므로 모델 로딩에 30~60초, 토큰 생성에 초당 1~5토큰 정도 소요됩니다.  
`MAX_TOKENS` 값을 줄여서 빠르게 테스트할 수 있습니다.

```python
MAX_TOKENS = 64  # 빠른 테스트용
```

### 생성된 코드 품질이 낮은 경우

- 프롬프트에 더 많은 컨텍스트를 추가하세요 (헤더, 관련 구조체 정의 등)
- `TEMPERATURE`를 낮추세요 (0.1 또는 0.0)
- 영어 주석을 사용하세요
