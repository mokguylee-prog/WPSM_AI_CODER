# 양자화 옵션

VRAM이 부족할 때 양자화로 모델 크기를 줄입니다. 품질 손실은 미미합니다.

---

## 4-bit 양자화 (NF4) - 가장 적극적인 절약

```python
import torch
from transformers import AutoTokenizer, AutoModelForCausalLM, BitsAndBytesConfig

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,  # 추가 압축
    bnb_4bit_quant_type="nf4"        # 가장 정확한 4-bit 타입
)

model = AutoModelForCausalLM.from_pretrained(
    "bigcode/starcoder2-7b",
    quantization_config=bnb_config,
    device_map="auto"
)
```

| 모델 | 원래 VRAM | 4-bit 후 |
|------|-----------|---------|
| 3B | 6.3 GB | ~2 GB |
| 7B | 14.6 GB | ~4 GB |
| 15B | 32 GB | ~9 GB |

---

## 8-bit 양자화 - 균형점

```python
bnb_config = BitsAndBytesConfig(
    load_in_8bit=True,
    bnb_8bit_compute_dtype=torch.bfloat16,
)

model = AutoModelForCausalLM.from_pretrained(
    "bigcode/starcoder2-7b",
    quantization_config=bnb_config,
    device_map="auto"
)
```

---

## bfloat16 (양자화 없음) - 최상의 품질

```python
model = AutoModelForCausalLM.from_pretrained(
    "bigcode/starcoder2-7b",
    torch_dtype=torch.bfloat16,
    device_map="auto"
)
```

## 품질 vs 속도 비교

| 모드 | 품질 | 속도 | VRAM |
|------|------|------|------|
| float32 | 최상 | 느림 | 2x |
| bfloat16 | 최상 | 빠름 | 1x (기준) |
| 8-bit | 좋음 | 중간 | 0.5x |
| 4-bit NF4 | 좋음 | 빠름 | 0.25x |

> C 코드 생성 목적으로는 4-bit도 충분히 좋은 품질을 제공합니다.
