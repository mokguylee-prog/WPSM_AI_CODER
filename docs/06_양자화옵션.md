# 양자화 옵션

## 현재 사용 방식: GGUF 사전 양자화

Sm_AICoder는 **미리 양자화된 GGUF 파일**을 사용합니다.
런타임에 양자화를 수행하지 않으며, HuggingFace에서 양자화된 모델을 직접 다운로드합니다.

---

## GGUF 양자화 포맷 비교

| 포맷 | 비트 | 파일 크기 (7B 기준) | 품질 | 속도 |
|------|------|-------------------|------|------|
| Q2_K | 2-bit | ~2.5GB | 낮음 | 매우 빠름 |
| Q4_K_M | 4-bit | ~4.4GB | **좋음 (권장)** | **빠름** |
| Q5_K_M | 5-bit | ~5.3GB | 매우 좋음 | 보통 |
| Q6_K | 6-bit | ~6.0GB | 거의 원본 | 느림 |
| Q8_0 | 8-bit | ~7.5GB | 원본 급 | 가장 느림 |

> 현재 프로젝트 기본값은 **Q4_K_M**입니다. C 코드 생성 목적으로 충분한 품질을 제공합니다.

---

## 사용 중인 모델별 양자화 크기

| 모델 | 양자화 | 파일 크기 | RAM 사용량 |
|------|--------|----------|-----------|
| Qwen2.5-Coder-7B-Instruct | Q4_K_M | ~4.4GB | ~5GB |
| Qwen2.5-Coder-1.5B-Instruct | Q4_K_M | ~1.0GB | ~1.5GB |
| DeepSeek-Coder-6.7B-Instruct | Q4_K_M | ~3.8GB | ~4.5GB |

> RAM 사용량 = 모델 크기 + 컨텍스트 버퍼 (~0.5~1GB)

---

## 다른 양자화 모델 사용하기

HuggingFace에서 원하는 양자화 포맷의 GGUF 파일을 직접 다운로드할 수 있습니다.

```bash
# 예: Q5_K_M (더 높은 품질)
huggingface-cli download Qwen/Qwen2.5-Coder-7B-Instruct-GGUF \
    --include "qwen2.5-coder-7b-instruct-q5_k_m.gguf" \
    --local-dir ./Sm_AICoder/models/gguf

# 예: Q2_K (최소 메모리, 빠른 테스트)
huggingface-cli download Qwen/Qwen2.5-Coder-7B-Instruct-GGUF \
    --include "qwen2.5-coder-7b-instruct-q2_k.gguf" \
    --local-dir ./Sm_AICoder/models/gguf
```

서버는 `Sm_AICoder/models/gguf/` 폴더에서 **가장 큰 GGUF 파일**을 자동 선택합니다.
원하는 모델만 폴더에 남겨두면 됩니다.

---

## 양자화 선택 가이드

| RAM | 권장 양자화 | 권장 모델 |
|-----|------------|----------|
| 8GB | Q4_K_M | Qwen2.5-Coder-1.5B (경량) |
| 16GB | Q4_K_M | Qwen2.5-Coder-7B (기본 권장) |
| 32GB+ | Q5_K_M ~ Q8_0 | Qwen2.5-Coder-7B (고품질) |

---

## (참고) GPU 사용 시 GGUF 양자화 효과

GPU를 사용하더라도 GGUF 양자화 모델을 그대로 사용합니다.
VRAM 요구량도 양자화에 비례하여 줄어듭니다.

| 모델 (7B) | Q4_K_M VRAM | Q8_0 VRAM | 비양자화 VRAM |
|-----------|-------------|-----------|-------------|
| Qwen2.5-Coder-7B | ~4.5GB | ~7.5GB | ~14GB |
