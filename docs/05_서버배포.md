# 서버 배포 가이드

## 현재 구성: FastAPI + llama-cpp-python (CPU 전용)

Sm_AICoder는 FastAPI 서버 하나로 동작하며, 별도의 추론 엔진 설치 없이 사용할 수 있습니다.

---

## 서버 시작

```powershell
# 방법 1: PowerShell 스크립트 (권장)
.\\start_server.ps1

# 방법 2: 백그라운드 런처 (PID 관리, 헬스체크 대기)
venv\Scripts\python.exe server\server.py

# 방법 3: 포그라운드에서 직접 실행
venv\Scripts\python.exe server\scripts\api_server.py
```

서버가 `http://localhost:8888`에서 실행됩니다.

---

## 서버 종료

```powershell
.\\stop_server.ps1

# 또는 수동 (server/server.pid 파일에서 PID 확인 후)
taskkill /PID <pid> /F
```

## 제공 엔드포인트

| 메서드 | 경로 | 설명 |
| ------ | ---- | ---- |
| GET | `/` | 웹 대시보드 (HTML) |
| GET | `/health` | 서버·모델 상태 확인 |
| POST | `/generate` | 단발성 코드 생성 |
| POST | `/chat` | 다중 턴 대화 |
| GET | `/stats` | 통계 + 최근 요청 이력 |
| GET | `/logs/download` | 요청 로그 파일 다운로드 |

> 자세한 API 명세는 [API_SPEC.md](API_SPEC.md)를 참고하세요.

---

## 서버 설정값

| 항목 | 기본값 | 설명 |
| ---- | ------ | ---- |
| `PORT` | `8888` | 수신 포트 |
| `N_CTX` | `4096` | 컨텍스트 길이 (토큰) |
| `N_THREADS` | `8` | CPU 스레드 수 |
| `N_GPU_LAYERS` | `0` | GPU 오프로드 레이어 (CPU 전용 = 0) |
| `MODEL_DIR` | `Sm_AICoder/models/gguf` | GGUF 파일 탐색 경로 |

> 설정을 변경하려면 `server/scripts/api_server.py` 상단의 변수를 직접 수정하세요.

## 동작 확인

```bash
# 헬스체크
curl http://localhost:8888/health
# {"status":"ok","model":"qwen2.5-coder-7b-instruct-q4_k_m.gguf"}

# 코드 생성 테스트
curl -X POST http://localhost:8888/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "C 언어로 Hello World 작성해줘", "max_tokens": 256}'
```

## GPU 가속 (선택사항)

GPU가 있는 경우 `server/scripts/api_server.py`에서 `N_GPU_LAYERS`를 올려 추론 속도를 높일 수 있습니다.

```python
# 예: GPU에 33개 레이어 오프로드
N_GPU_LAYERS = 33
```

llama-cpp-python GPU 빌드가 필요합니다:

```bash
# CUDA 12.x
pip install llama-cpp-python --extra-index-url https://abetlen.github.io/llama-cpp-python/whl/cu121

# 또는 사전 빌드 바이너리
pip install llama-cpp-python --prefer-binary
```

## 클라이언트 연동

```text
사용자
  │
  ├─ GUI 클라이언트 (client/gui_client.py)
  │     ├── GET  /health  → 상태 확인
  │     └── POST /chat    → 대화 전송
  │
  ├─ CLI 클라이언트 (client/client.py)
  │     ├── GET  /health  → 접속 확인
  │     └── POST /chat    → 대화 전송
  │
  └─ 브라우저 (localhost:8888)
        ├── GET  /         → 대시보드
        ├── GET  /stats    → 통계 폴링
        └── GET  /logs/download → 로그 다운로드
```
