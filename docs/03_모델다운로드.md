# 모델 다운로드

## 다운로드 방법 비교

| 방법 | 장점 | 단점 |
|------|------|------|
| Python 자동 다운로드 | 간편 | 캐시 위치 제어 어려움 |
| huggingface-cli | 경로 지정 가능, 재시도 지원 | CLI 별도 설치 |
| git lfs | 전체 저장소 복제 | 느림, 용량 큼 |
| GGUF (llama.cpp용) | CPU에서도 빠름 | 변환 필요 |

---

## 방법 1: Python 자동 다운로드 (가장 간편)

처음 실행 시 자동으로 다운로드됩니다.

```python
from transformers import AutoTokenizer, AutoModelForCausalLM

checkpoint = "bigcode/starcoder2-7b"
tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForCausalLM.from_pretrained(checkpoint)
```

기본 캐시 위치:
- Windows: `C:\Users\<username>\.cache\huggingface\hub\`
- Linux: `~/.cache/huggingface/hub/`

캐시 위치 변경:
```python
model = AutoModelForCausalLM.from_pretrained(
    checkpoint,
    cache_dir="D:/models/starcoder2"
)
```

## 방법 2: huggingface-cli (권장)

```bash
# 설치
pip install huggingface_hub[cli]

# 로그인
huggingface-cli login

# 3B 모델 다운로드 (~10GB)
huggingface-cli download bigcode/starcoder2-3b \
    --local-dir ./models/starcoder2-3b

# 7B 모델 다운로드 (~15GB)
huggingface-cli download bigcode/starcoder2-7b \
    --local-dir ./models/starcoder2-7b

# 15B 모델 다운로드 (~50GB)
huggingface-cli download bigcode/starcoder2-15b \
    --local-dir ./models/starcoder2-15b
```

진행률 표시와 함께 다운로드되며, 중단 후 재시도 가능.

## 방법 3: GGUF 모델 (CPU/저사양 GPU용)

llama.cpp와 함께 사용하는 경량 포맷:

```bash
# 3B GGUF (Q4_K_M 양자화, ~2GB)
huggingface-cli download second-state/StarCoder2-3B-GGUF \
    --include "starcoder2-3b-Q4_K_M.gguf" \
    --local-dir ./models/gguf

# 7B GGUF (Q4_K_M 양자화, ~4GB)
huggingface-cli download second-state/StarCoder2-7B-GGUF \
    --include "starcoder2-7b-Q4_K_M.gguf" \
    --local-dir ./models/gguf

# 15B GGUF (Q4_K_M 양자화, ~9GB)
huggingface-cli download second-state/StarCoder2-15B-GGUF \
    --include "starcoder2-15b-Q4_K_M.gguf" \
    --local-dir ./models/gguf
```

## 방법 4: git lfs

```bash
git lfs install
git clone https://huggingface.co/bigcode/starcoder2-7b ./models/starcoder2-7b
```

---

## 다운로드 후 로컬 경로로 로딩

```python
from transformers import AutoTokenizer, AutoModelForCausalLM

local_model_path = "./models/starcoder2-7b"

tokenizer = AutoTokenizer.from_pretrained(local_model_path)
model = AutoModelForCausalLM.from_pretrained(
    local_model_path,
    device_map="auto",
    torch_dtype="bfloat16"
)
```
