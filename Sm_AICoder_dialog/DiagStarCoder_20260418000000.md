# Session Log - 2026-04-18

## Context

This session covered two major changes:

1. Defining an evaluation model for agent mode behavior based on `image/진행방향.md`.
2. Extending the GUI client with VS Code-like project navigation and clipboard/image attachment workflows.

## Work Completed

### 1) `image/진행방향.md` rewrite

The file was rewritten to make agent behavior explicit and testable.

Changes included:

- Separated responsibilities into `평가 AGENT` and `개발 AGENT`.
- Added a scoring rubric for agent behavior.
- Added checklist sections for each role.
- Added an operation flow:
  - receive request
  - evaluate behavior against the guide
  - implement changes
  - re-evaluate
- Added forbidden behaviors and usage examples.

The intent was to make agent-mode operation measurable rather than informal.

### 2) Client UI and workflow changes

The GUI client in `client/gui_client.py` was extended so the client command workflow can reference a selected working folder.

Implemented:

- `OpenFolder` button and project folder selection.
- File and folder creation buttons.
- A folder tree panel styled more like VS Code, with expand/collapse support.
- Folder path display.
- File selection and attachment handling.
- Clipboard paste support for:
  - text
  - file paths from the clipboard
  - clipboard images saved into the selected folder under `.sm_aicoder_assets`
- A preview panel:
  - image thumbnails for supported image types
  - textual preview for files
  - folder summary for selected folders
- Prompt context injection:
  - selected OpenFolder path
  - visible folder tree entries
  - automatic summaries of some files
  - attachment paths
- Agent mode `working_dir` was changed to use the selected OpenFolder.

### 3) Dependencies

`requirements.txt` was updated to add:

- `Pillow>=10.0.0`

This was needed for clipboard image handling and thumbnail previews.

### 4) Validation

Validation completed:

- `python -m py_compile client\gui_client.py`

This passed after the GUI changes.

- The prompt builder currently injects file list and file summaries from the selected folder so the agent can reference local project context.
